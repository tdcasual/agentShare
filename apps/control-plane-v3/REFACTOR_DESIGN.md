# Control Plane V3 重构设计方案

## 1. 需求分析

### 1.1 多语言需求
- **默认语言**: 中文 (zh-CN)
- **支持语言**: 中文、英文
- **切换方式**: Header 下拉菜单
- **覆盖范围**: 所有页面和组件

### 1.2 深色模式需求
- **风格**: 二次元 + 深色，但**不要霓虹灯**
- **色调**: 柔和暗色系 + 低饱和度粉色
- **对比度**: 保持可读性，避免刺眼
- **元素处理**:
  - 背景: 深灰紫/深粉色系，非纯黑
  - 卡片: 暗色 + 粉色边框（柔和发光）
  - 强调色: 降低饱和度的人类蓝、智能体绿
  - 文字: 灰白色系，避免纯白刺眼

### 1.3 页面重构需求
需要重构为二次元风格的页面:
- `/login` - 登录页
- `/setup` - 初始化页  
- `/tokens` - Token 管理页
- `/reviews` - 审核队列页

需要适配多语言和深色模式的页面:
- `/` - Hub 首页
- `/identities` - 身份浏览
- `/spaces` - 协作空间
- `/tasks` - 任务看板
- `/assets` - 资产页面
- `/settings` - 设置中心
- `/marketplace` - 市场

## 2. 技术方案

### 2.1 多语言方案 (next-intl)

```typescript
// 结构
src/
  i18n/
    config.ts          # i18n 配置
    messages/
      zh-CN.json       # 中文
      en.json          # 英文
    navigation.ts      # 路由处理
  components/
    language-switcher.tsx  # 语言切换组件
```

**配置要点**:
- 默认语言: zh-CN
- 路由前缀: /zh-CN, /en
- 回退策略: 中文 → 英文 → key

### 2.2 深色模式方案 (next-themes)

**颜色系统设计**:

```css
/* 浅色模式 (现有) */
--kw-bg: #FFF5F7
--kw-surface: #FFFFFF
--kw-primary: #FF69B4
--kw-text: #4A4A4A

/* 深色模式 (新增) - 柔和暗色系 */
--kw-bg-dark: #1A1A2E        /* 深紫灰，非纯黑 */
--kw-surface-dark: #252540   /* 稍亮的卡片背景 */
--kw-primary-dark: #E891C0   /* 降饱和度粉色 */
--kw-text-dark: #E8E8EC      /* 柔和白 */
--kw-text-muted-dark: #9CA3AF

/* Human/Agent 强调色 (降低饱和度) */
--kw-human-dark: #6B9AC4     /* 柔和蓝 */
--kw-agent-dark: #7FB883     /* 柔和绿 */
```

**设计原则**:
1. 避免纯黑 (#000000)，使用深紫/深蓝灰
2. 粉色主色降低饱和度，避免霓虹感
3. 边框发光效果柔和（opacity 0.1-0.2）
4. 阴影使用粉色/紫色混合，扩散度大但透明度低

### 2.3 组件重构方案

#### 登录页 (/login)
**当前问题**: 功能性强，缺乏二次元装饰
**改进**:
- 添加浮动装饰背景（樱花、星星）
- 卡片使用玻璃拟态效果
- 输入框添加粉色发光边框
- 添加吉祥物角色（可选）

#### 初始化页 (/setup)
**当前问题**: 表单布局，信息密度高
**改进**:
- 步骤指示器（装饰性）
- 分步骤表单或折叠区域
- 添加引导性插图区域
- 保持信息清晰的同时增加可爱元素

#### Token 管理页 (/tokens)
**当前问题**: 数据表格感强
**改进**:
- Agent 卡片网格布局（替代列表）
- Token 使用徽章/标签样式
- 添加统计卡片（圆角大、阴影柔和）
- 操作按钮使用图标 + 文字

#### 审核队列页 (/reviews)
**当前问题**: 列表形式，审批操作生硬
**改进**:
- 审核项使用卡片式布局
- 添加状态图标（可爱风格）
- Approve/Reject 按钮使用不同形状
- 添加资源类型图标
